# 测试指南

## CSV 功能测试

### 1. CSV 上传测试

**测试文件**: `test-pattern.csv`（项目根目录）

**测试步骤：**
1. 启动开发服务器：`npm run dev`
2. 打开浏览器访问应用
3. 点击或拖拽 `test-pattern.csv` 到上传区域
4. 验证是否显示 3x3 的彩色网格
5. 检查是否显示成功 Toast 通知

**预期结果：**
- ✅ 显示 3x3 网格
- ✅ 颜色正确：粉红、青绿、黄色
- ✅ 显示"CSV 文件加载成功！"通知
- ✅ 颜色统计显示 3 种颜色

### 2. CSV 导出测试

**测试步骤：**
1. 上传图片或 CSV 生成图纸
2. 点击"导出 CSV"按钮
3. 检查下载的 CSV 文件
4. 重新上传该 CSV 文件

**预期结果：**
- ✅ 成功下载 CSV 文件
- ✅ 显示成功动画（绿色圆圈 + 对勾）
- ✅ CSV 格式正确（x,y,color,code）
- ✅ 重新上传后图纸一致

### 3. CSV 模板下载测试

**测试步骤：**
1. 点击"下载 CSV 模板"链接
2. 打开下载的模板文件
3. 修改模板数据
4. 上传修改后的 CSV

**预期结果：**
- ✅ 成功下载模板文件
- ✅ 模板格式正确
- ✅ 修改后可正常上传

### 4. CSV 错误处理测试

**测试场景 A：缺少必需列**
```csv
x,y
0,0
1,1
```

**预期结果：**
- ✅ 显示错误 Toast："CSV 文件必须包含 x, y, color 列"

**测试场景 B：无效颜色**
```csv
x,y,color
0,0,#INVALID
```

**预期结果：**
- ✅ 控制台警告："颜色 #INVALID 不在调色板中"
- ✅ 自动使用最接近的颜色

**测试场景 C：空文件**
```csv
x,y,color
```

**预期结果：**
- ✅ 显示错误 Toast："CSV 文件中没有有效的像素数据"
