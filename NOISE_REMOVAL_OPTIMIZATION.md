# 一键去杂色功能优化说明

## 优化内容

### 1. 参数范围调整 ✅
**变更**：
- 原范围：0.1% - 1.0%
- 新范围：**0.0% - 1.0%**

**原因**：
- 有些极少数量的杂色占比趋近于 0.0%
- 例如：10000 颗珠子中只有 1-2 颗的颜色，占比仅 0.01%-0.02%
- 支持 0.0% 可以移除这些极少数量的杂色

**使用建议**：
- 0.0%：移除数量极少的杂色（1-2 颗）
- 0.1%：移除数量很少的杂色（10 颗以下，基于 10000 颗总数）
- 0.5%：移除数量较少的杂色（50 颗以下）
- 1.0%：移除数量少的杂色（100 颗以下）

---

### 2. UI/UX 设计统一 ✅
**变更**：
- 原设计：滑块输入
- 新设计：**数字输入框**

**改进点**：
- 与"网格尺寸"和"颜色合并阈值"保持一致的交互方式
- 支持精确输入小数值（如 0.3%）
- 显示范围提示：最小 0.0%，最大 1.0%
- 自动限制输入范围，防止超出

**输入特性**：
- 支持小数点后一位（如 0.5）
- 支持键盘输入和上下箭头调整
- 输入超出范围会自动修正

---

### 3. 算法优化 ✅

#### 3.1 阈值计算优化
**问题**：
- 原算法使用 `Math.ceil(totalBeads * thresholdPercent)`
- 即使阈值很小，也至少是 1 颗珠子
- 导致无法精确移除极少数量的颜色

**解决方案**：
```typescript
// 原算法
const threshold = Math.ceil(totalBeads * thresholdPercent); // 至少为 1

// 新算法
const threshold = totalBeads * thresholdPercent; // 精确浮点数计算
if (count < threshold) { // 使用小于而不是小于等于
  noiseColors.add(hex);
}
```

**效果**：
- 0.0% 阈值可以移除数量为 0 的颜色（理论上不存在）
- 0.01% 阈值可以移除数量 < 1 的颜色（实际上是 1 颗）
- 更精确的杂色识别

#### 3.2 智能颜色替换
**问题**：
- 原算法直接使用欧氏距离找最近颜色
- 可能导致替换后的颜色与周围不协调

**解决方案**：
```typescript
// 优先使用相邻单元格的颜色
const neighborColor = findNeighborColor(newGrid, row, col, noiseColors);

if (neighborColor) {
  // 使用相邻颜色（更自然）
  newGrid[row][col].paletteColor = neighborColor;
} else {
  // 如果没有相邻颜色，使用最近颜色
  const nearestColor = findNearestColor(cell.paletteColor, validColors);
  newGrid[row][col].paletteColor = nearestColor;
}
```

**优势**：
- 优先查找上下左右四个方向的相邻单元格
- 如果相邻单元格有非杂色，直接使用该颜色
- 使替换后的图案更自然、更连贯
- 避免出现孤立的颜色点

#### 3.3 边界情况处理
**新增**：
```typescript
// 如果所有颜色都是杂色，保留数量最多的颜色
if (validColors.length === 0) {
  const maxColor = findMostFrequentColor(colorCounts);
  noiseColors.delete(maxColor);
  validColors.push(palette.find(c => c.hex === maxColor)!);
}
```

**作用**：
- 防止所有颜色都被移除的极端情况
- 确保至少保留一种颜色

## 使用示例

### 示例 1：移除极少数量的杂色
```
场景：10000 颗珠子的图案
- 红色：5000 颗（50%）
- 蓝色：4000 颗（40%）
- 绿色：990 颗（9.9%）
- 黄色：8 颗（0.08%）
- 紫色：2 颗（0.02%）

设置阈值：0.1%
阈值珠子数：10000 × 0.1% = 10 颗

结果：
- 黄色（8 颗）被移除 ✓
- 紫色（2 颗）被移除 ✓
- 其他颜色保留
```

### 示例 2：智能颜色替换
```
原图案：
R R R R R
R Y R R R  ← Y 是杂色（黄色）
R R R R R

替换后：
R R R R R
R R R R R  ← Y 被替换为相邻的 R（红色）
R R R R R

而不是：
R R R R R
R B R R R  ← 如果只用欧氏距离，可能被替换为 B（蓝色）
R R R R R
```

## 技术细节

### 算法复杂度
- 时间复杂度：O(M × N)，M 和 N 为网格尺寸
- 空间复杂度：O(M × N)，需要深拷贝网格

### 性能优化
- 使用 Set 存储杂色，查找时间 O(1)
- 相邻颜色查找只检查 4 个方向
- 避免重复计算颜色统计
